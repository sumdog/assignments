13.3 Consider the following I/O scenarios on a single-user PC:

For Each of these scenarios, would you design the operating system to use buffering, spooling, caching or a combination?  Would you use polled I/O or interrupt driven I/O? Give your reasons for your choices.

a.       A mouse used with a GUI

In this case buffering could be needed to record mouse movement when higher priority operations are taking place.  Interrupt driven I/O would be optimal because polled I/O would waste time by polling when higher priority operations are taking place.

b.      A tape drive on a multitasking operating system

In this case I would use a combination because buffering could be used when reading from the tape, which could require long read times and caching could be used for faster access to memory on the tape drive.  Spooling would be helpful because a tape drive would not be able to service request of multiple concurrent applications.  I would use interrupt driven I/O because it would have better performance than polled I/O in this case.

c.       A disk drive containing user files

Buffering could be used when writing and reading files from the disk.  Cahing can be used to improve performance of the disk.  Interrupt driven I/O is better in this case because it is better for devices with a slow transfer rate.

d.      A graphics card with direct bus connection, accessible through memory mapped I/O.

Since the device using memory mapped I/O it will not need polled I/O or interrupt driven I/O.  Since it should be quite fast it will not need cache or spooling.  However, buffering could be helpful for servicing multiple concurrent calls to the graphics card.
