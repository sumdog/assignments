  • 6.3 What is the meaning of the term busy waiting? What other kinds of waiting are there in an operating system? Can busy waiting be avoided altogether? Explain your answer.
    • In busy waiting, a process blocks and holds the processor until it waits for an instruction to complete, such as making a request from memory or disk. An alternative is for the task to yield the processor back to the operating system so that it can schedule another task to run. Typically if the task request some type of blocking I/O (memory or disk) and interrupt is sent to the operating system when the operation is complete and it will load that process back into the running state so it can complete its operation.
  • 6.4 Explain why spinlocks are not appropriate for single-processor systems yet are often used in multiprocessor systems.
    • A spinlock is essentially a while look that keeps looping until a condition is met. They're great for quick tasks with minimal wait. For long task, the spin locks will continually be swapped out and in by the scheduler, essentially polling for a resource instead of waiting for an interrupt. On a single processor system, this can be very costly since only one process can run at a time and the process running the spinlock must be swapped in each time to poll. On a multi-processor systems, the spinning thread can be moved to another processor. Spinlocks can be avoided if there is a hardware based interrupt support that prevents the need for polling.
