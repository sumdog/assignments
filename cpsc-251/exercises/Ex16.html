  • 14.6 Discuss the strengths and weaknesses of implementing an access matrix using capabilities that are associated with domains.
    • In this implementation a capability list for a domain is a list of objects together with the operations allowed on those object. Possession of a capability on the capability list means that access is allowed. However the capability list is never directly accessible to a process executing in that domain. Rather the capability list itself is a protected object, maintained by the operating system and accessed by the user only indirectly. With all the capabilities secure, the objects they protect are also secure against unauthorized access.
    • A weakness is that access-rights information for a particular domain is not localized, which means that determining the set of access rights for each domain is difficult. Also, every access to the object must be checked, requiring a search of the access list. In a large system with long access lists, this search can be time consuming.
