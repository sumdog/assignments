  • 15.3 The list of all passwords is kept within the operating system. Thus, if a user manages to read this list, password protection is no longer provided. Suggest a scheme that will avoid this problem. (Hint: Use different internal and external representations.)
    • Linux solves this problem by keeping a file called /etc/passwd which stores basic user data such as usernames, home directories, etc. and another file called /etc/shadow which stores encrypted password and is only readable by the root user which authentication programs run as. Other solutions include using an LDAP directory which contains a database of user information and encrypted password.
  • 15.5 An experimental addition to UNIX allows a user to connect a watchdog program to a file. The watchdog is invoked whenever a program requests access to the file. The watchdog then either grants or denies access to the file. Discuss two pros and two cons of using watchdogs for security.
    • A watchdog program would be able to tell whether access to a file was permitted, or if the file was attempting to be accessed illegally. If the access was not allowed the watchdog could alert the system of the type of false access and tell the system how to respond.
    • A problem is that some watchdog systems would not be able to identify new methods of intrusion. Also the watchdog may generate false positives and not allow access to the file when it should be, or it may produce false negatives and allow access to the file when it shouldn't have been
  • 15.7 Discuss a means by which managers of systems connected to the Internet could have designed their systems to limit or eliminate the damage done by a worm. What are the drawbacks of making the change that you suggest?
    • There are a multitude of things a system administration can do. First and foremost setting up both software and hardware firewalls to only allow incoming connections on the services that will be used on the server. The disadvantage to this is that any new services require modifications to firewall tables before they can be accessed.
    • Next, the services that are accessible to the outside world should be hardened meaning that they should not be allowed to run as root (run as underprivileged). If it is possible, they should also be run either in a chroot environment or on a system with SELinux policies enabled. The disadvantage to SELinux is that granting the service access to other parts of the system can become a hassle. Chroot environments can easily become an administration nightmare when it comes to security updates and propagating them through each of the individual chroots.
