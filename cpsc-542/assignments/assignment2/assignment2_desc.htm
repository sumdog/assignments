DTD – We used string data-type for all data.  JavaScript could have been used to check the data types before submission; however, JavaScript has a chance of not working in all browsers, so we avoided it.  We used many attributes for our nodes and more work for the DTD to validate the data instead of having tons of simple nodes.

 

<?xml version="1.0"?>
  <!DOCTYPE application [
  <!ELEMENT application (information,courses,transfercourses,exam,totalhours,signatures)>
  <!ELEMENT information (name,utcid,address,address?,phone,degreetype,major,concentration,graduationdate,utcemail)>
  <!ELEMENT name (#PCDATA)>
  <!ELEMENT utcid (#PCDATA)>
  <!ELEMENT address (#PCDATA)>
  <!ATTLIST address line (1|2) "1">
  <!ELEMENT phone (#PCDATA)>
  <!ELEMENT degreetype (#PCDATA)>
  <!ATTLIST degreetype type (Master|Specialist|Doctor) #REQUIRED>
  <!ELEMENT major (#PCDATA)>
  <!ELEMENT concentration (#PCDATA)>
  <!ELEMENT graduationdate (#PCDATA)>
  <!ELEMENT utcemail (#PCDATA)>
  <!ELEMENT courses (course*)>
  <!ELEMENT course EMPTY>
  <!ATTLIST course term CDATA "">
  <!ATTLIST course hours CDATA "">
  <!ATTLIST course grade CDATA "">
  <!ATTLIST course department CDATA "">
  <!ATTLIST course course CDATA "">
  <!ATTLIST course title CDATA "">
  <!ELEMENT transfercourses (course*)>
  <!ELEMENT exam EMPTY>
  <!ATTLIST exam required (yes|no) "yes">
  <!ELEMENT totalhours (#PCDATA)>
  <!ELEMENT signatures (sig*)>
  <!ELEMENT sig (#PCDATA)>
  <!ATTLIST sig type (applicant|advisor|director|dean) #IMPLIED>
  <!ATTLIST sig date CDATA #IMPLIED>
]> 

XML – The node “application” was the main wrapper for “information”, “courses”, “transfercourses”, “exam”, “totalhours”, and “signatures”. 

“information” contained 9 different nodes which were “name”, “utcid”, “address” (with attribute “line”… being only “1” or “2”), “degreetype” (with attribute “type”… being either “Master, Specialist, or Doctor”), “concentration”, “graduationdate”, and “utcemail”.

“courses” and “transfercourses” contain 5 nodes of “course”.  “course” contains the 6 attributes “term”, “hours”, “grade”, “department”, “course”, and “title”.

“exam” containes the attribute “required” (with the attribute being either “yes” or “no”).

“totalhours” is just a simple node of data describing the total hours of all of the courses combined.

“signatures” contains 4 nodes of “sig”.  “sig contains 2 attributes, one of  them being “type” (which can be either “applicant”, “advisor”, “director”, or “dean”), and the second being a simple “date” attribute.
